一、Git Introduction

1.1、 Git是一个免费的开源 分布式版本控制软件，主要用于管控项目。操作Git一般有2种方式：Git Bash命令行操作，或者Git GUI工具进行操作，其中最优秀的2款工具是TortoiseGit(小乌龟)和SourceTree(更强大)。之后会详细讲解如何使用。
1.2、同时衍生除了很多基于GIt的代码托管平台，如GitHub(国外,现被微软收购)、Gitee(码云国内)、Conding(腾讯代理)、GitLab(企业常用)。

二、 Git的三个区（working,staging、local）->重点理解

2.1、 Workspace：工作区：即项目所在的文件夹
2.2、 Index / Stage：暂存区：简单理解为.git文件下的Index文件
2.3、 Repository：仓库区（或本地仓库):本地的版本库,简单理解为.git文件夹
2.4、 Remote：远程仓库

四、自己整理的常用命令

Git CheatSheet

一、Git Basic

$ git clone <url>      -> 克隆远程仓库代码到本地
$ git add .            -> 添加所有文件到暂存区
$ git status           -> 检测文件状态
$ git commit -m "xxxx" -> 添加文件到本地仓库

$ git pull             -> 拉取远程代码并合并到当前分支
$ git push             -> 推送本地代码到远程(先pull后push，同时确保当前工作区是clean)

二、Git config

$ cat ~/.gitconfig 查看配置

//首次 commit 必须向 Git 自报家门,--global 参数如果在单独项目文件下设置，可以不必加。name 和 email
$ git config --global user.name "Your Name"
$ git config --global user.email "you@example.com"

三、Git log 查看提交日志

 $ git log                              ->列出当前分支commit记录日志(只能是当前commit的以往commit历史记录)
 $ git log -n                           ->列出最近n次commit记录
 $ git log -p -2                        ->列出最近2次提交日志并显示出两者差异
 $ git log --oneline --graph --decorate ->以较好的方式(单行,图形化)列出提交日志

四、Git Branches 分支操作

 $ git branch <name>:创建分支
 $ git checkout <name>:切换分支
 $ git checkout -b <name>:切换并创建分支
 $ git branch -d/D <name>:删除分支

 $ git merge <name>  ->合并当前分支,注意一定要切换到当前分支
 $ git branch        ->列出所有本地分支
 $ git branch -r     ->列出所有远程分支
 $ git branch -a     ->列出本地和远程所有分支

五、Git Reset 代码回滚

// Git 将"文件管理"分为2个区域,4个状态:原内容->修改过的内容->暂存区-->版本库(本地仓库),利用reset可以在任意区域切换

$ git reset --soft <版本号> ->版本库回滚到暂存区
$ git reset HEAD <file1> <file2> ->文件从暂存区回滚到修改过的内容

$ git reflog  -> 列出历史操作记录
$ git reset --hard <版本号>  ->从版本库回滚到工作区(任意一次commit记录)

六、Remote Resposity

$ git init       ->初始化本地仓库
$ git remote -v ->查看关联的远程仓库
$ git remote add origin(远程仓库名，一般命名为 origin) [url]:关联远程仓库

$ git push -u origin master ->本地仓库码 master 分支与远程 master 建立追踪关系

[注意]
  以上情况，git init 初始化的仓库和远程仓库 origin master 分支默认并未建立追踪关系,git clone 下的代码，则并不需要这样操作,git clone 默认会将远程所有分支拉取过来,但如果此时 git branch 只会有 master 分支，需要使用 git checkout <name>切换到某个分支，并建立 track 关系，此时本地就拥有该分支了。
